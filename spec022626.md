壹、 系統功能矩陣表（Function List）
本矩陣表依據計畫目標，分為六大模組，明確定義「核心要求」與「擴充項目」，供投標廠商評估開發規模與技術架構。
模組一：安全沙盒與基礎推論架構（核心要求）
此模組為系統運作之底層基礎，確保算力充沛且機敏資料絕對安全。
功能編號	功能模組/項目	功能描述規格要求	類別
1.1	地端 GPU 資源池管理	系統需支援地端 NVIDIA GPU 或同等級算力設備之資源調度（如支援 vGPU 或 MIG 技術），可依據 Agent 負載動態分配 VRAM。	核心
1.2	實體/邏輯網路隔離	建置符合本署資安規範之封閉型沙盒（Sandbox）環境，確保推論過程、上傳檔案與生成結果皆不連接公網（Air-gapped 或嚴格 VLAN 隔離）。	核心
1.3	大語言模型 (LLM) 佈署	需於沙盒內完成開源大型語言模型（如 Llama 3 或同等級、支援繁體中文與醫學領域之開源模型）之落地佈署與微調（Fine-tuning）環境建置。	核心
1.4	系統災備與高可用性 (HA)	具備資料庫定期備份與系統故障自動切換機制，確保 RTO（復原時間目標）< 4小時，RPO（資料復原點目標）< 1小時。	核心

針對歷史審查紀錄與廠商送件資料進行 PII/PHI 與營業秘密的遮蔽。
功能編號	功能模組/項目	功能描述規格要求	類別
2.1	多格式文本解析與 OCR	支援 PDF、Word、Excel 及掃描圖檔的文字萃取。圖檔需具備高精準度光學字元辨識（OCR），支援繁中、英文及常見醫學符號。	核心
2.2	實體命名辨識 (NER) 遮蔽	利用 NLP 技術自動辨識並遮蔽：廠商名稱、負責人、地址、電話、專利號碼、受試者個資等機敏欄位。	核心

參、 針對特定醫材（SaMD）的 Agentic AI 審查邏輯協作範例
有別於傳統 RAG 只能「一問一答」，Agentic AI 具備「思考、拆解、協作、彙整」的能力。以下以一件**「基於深度學習的肺結節輔助診斷軟體 (AI-CAD SaMD)」**的上市前查驗登記案為例，展示多智能體的協作審查邏輯（Workflow）。
1. 參與協作的 AI Agent 角色定義
Meta-Agent (總控官)：負責解析廠商送件摘要（STED）、分配任務、協調衝突並生成最終官方公文草稿。
Agent_Regulatory (法規檢索專家)：專精於《醫療器材軟體查驗登記技術指引》、軟體分級（Class II/III）判斷。
Agent_CyberSecurity (資安與確效專家)：專精於 IEC 62304（軟體生命週期）、FDA 網路安全指引、SBOM（軟體物料清單）解析。
Agent_Clinical (臨床績效專家)：具備圖表解析能力與統計基礎，負責核對 Sensitivity、Specificity 等臨床評估報告數據。
2. 多智能體協作流程 (Workflow Steps)
步驟一：案件受理與任務拆解 (Intake & Task Disassembly)
觸發條件：審查員在系統點擊「啟動 AI 輔助初審」。
Meta-Agent 動作：
掃描廠商上傳的 5 份核心 PDF（包含：軟體描述、網路安全報告、軟體確效報告、臨床試驗總結報告、說明書）。
思考鏈 (Chain of Thought)：「偵測到產品為利用 AI 分析 X 光影像之電腦輔助診斷軟體，屬 SaMD 範疇，風險等級可能為 Class II。我需要啟動法規、資安與臨床三個 Agent 進行並行審查。」
任務分派：
發送指令給 Agent_Regulatory：「確認此產品的預期用途是否符合 SaMD 定義，並檢核標籤說明書合規性。」
發送指令給 Agent_CyberSecurity：「審閱網路安全報告，尋找威脅建模（Threat Modeling）與漏洞修補計畫。」
發送指令給 Agent_Clinical：「從臨床總結報告中提取 ROC 曲線數據與 95% 信心區間，核對試驗人數是否符合原核准之計畫書。」
步驟二：平行審查與動態檢索 (Parallel Processing & RAG)
各專責 Agent 利用 RAG 檢索專屬向量庫，並對文件進行深度解析：
Agent_Regulatory 檢索向量庫中的「醫療器材標籤指引」，發現廠商的說明書中「未明確標示 AI 演算法的限制條件（例如：不適用於小於 3mm 的結節）」。
Agent_CyberSecurity 解析了 200 頁的軟體確效報告，發現缺少了近期的開源軟體漏洞掃描紀錄（SBOM 中引用的某個 Python 影像處理函式庫版本過舊）。
Agent_Clinical 透過表格解析工具讀取臨床數據，發現廠商宣稱的靈敏度為 95%，但附錄的統計檢定 P-value > 0.05，未達統計顯著性。
步驟三：跨 Agent 協調與防幻覺查核 (Conflict Resolution & Cross-Check)
Agent_Clinical 將「未達統計顯著性」的重大異常回報給 Meta-Agent。
Meta-Agent 啟動防護機制，要求 Agent_Clinical 必須提供原始出處。
Agent_Clinical 回覆：「根據檔案『附錄C_臨床統計數據.pdf』第 45 頁表 3，子群組分析中的 P-value 為 0.062，未滿足小於 0.05 之標準（Confidence Score: 96%）。」
此時，Agent_Regulatory 提出警告：「依據《AI 醫療軟體臨床評估指引》，若子群組未達顯著差異，需在仿單中加註『適用族群警語』。」
Meta-Agent 綜合兩者意見，確認這是一個重大審查缺失。
步驟四：彙整生成結構化審查草稿 (Draft Generation)
Meta-Agent 彙整所有子 Agent 的發現，依據本署標準的「補件通知函」格式，產出結構化草稿，呈現在雙視窗介面的右側供審查員核閱：
【AI 輔助審查初步意見草稿】
案件類型：SaMD 軟體醫療器材 (AI-CAD)
AI 綜合信心分數：88% (建議重點人工覆核臨床數據)
一、 臨床評估缺失 (優先級：高)
經查廠商所附之「附錄C_臨床統計數據.pdf」第 45 頁表 3，針對 65 歲以上族群之子群組分析，其 P-value 為 0.062，未達統計顯著性差異。依據《人工智慧/機器學習技術之醫療器材軟體臨床評估指引》，請廠商補充說明該子群組之臨床安全性與有效性，或於中文仿單之「警語及注意事項」中載明不適用於該特定族群。(來源：臨床數據 Agent，信心度 96%)
二、 網路安全與軟體確效缺失 (優先級：中)
依據上傳之「網路安全風險管理報告.pdf」第 112 頁之 SBOM 物料清單，貴公司採用之開源套件 (OpenCV v3.4.1) 包含已知未修補之 CVE 漏洞。請依據《醫療器材網路安全指引》，提供漏洞評估分析（Vulnerability Assessment）及未來上市後之軟體更新修補計畫。(來源：資安確效 Agent，信心度 92%)
三、 標籤與說明書缺失 (優先級：低)
中文仿單草案未詳述 AI 演算法之操作限制條件（如：影像解析度要求、設備相容性等）。請依規定補齊。(來源：法規 Agent，信心度 98%)
3. Human-in-the-loop (審查員介入與系統學習)
審查員介入：資深審查員檢閱 AI 草稿後，認為「三、標籤缺失」部分廠商其實在另一份「操作手冊.pdf」中有提及，只是未整合至仿單。
手動編修：審查員在介面上將第三點修改為：「請將『操作手冊.pdf』第 15 頁所述之設備相容性限制，整併更新至中文仿單草案中。」
回饋迴圈 (Feedback Loop)：系統後台紀錄此修改軌跡。未來的 Agent_Regulatory 在微調後，將學會「在指控仿單缺失前，必須先交叉比對操作手冊」。
肆、 結論與後續推動建議
本計畫導入之 Agentic AI 智慧輔助系統，徹底顛覆了過往「單向關鍵字檢索」的窠臼。透過安全沙盒保障機敏資料不外流、自動去識別化活化歷史資產，以及建置具備專家邏輯的多智能體協作架構 (Meta-Agent & Sub-Agents)，系統將能承擔繁重的法規比對與數據核實工作。
